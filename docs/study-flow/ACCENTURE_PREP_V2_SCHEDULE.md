### PROJECT 3 — Python Log Analyzer Script (Day 3, 2 hrs)
**Stack:** Python 3 + re (regex) + json + collections
**Purpose:** Maps to JD: "JSON parsing, regex, light scripting"

```python
# log_analyzer.py — what to build
# Input: a sample.log file with lines like:
# 2024-01-15 10:23:45 ERROR service=order-service msg="DB timeout" user=u123
# 2024-01-15 10:23:46 INFO  service=api-gateway msg="Request received"

# Script should:
# 1. Parse each line using regex: extract timestamp, level, service, message, user
# 2. Count errors per service
# 3. Find top 3 most common error messages
# 4. Output a summary JSON report
# 5. Flag any service with > 5 errors as "critical"
```

```
Steps:
1. Create sample.log with 30+ lines (mix of ERROR/WARN/INFO)
2. Write parser using re.search() with named groups
3. Use collections.Counter for frequency analysis
4. Output to report.json using json.dumps()
5. Add CLI argument: python log_analyzer.py --level ERROR --service order-service
6. Push to GitHub: repo name "python-log-analyzer"
```
**Resource:** https://docs.python.org/3/library/re.html (official — fastest reference)
**Interview talking point:** "I built a Python log analyzer using regex and JSON that identifies recurring error patterns — exactly what this triage role needs."
