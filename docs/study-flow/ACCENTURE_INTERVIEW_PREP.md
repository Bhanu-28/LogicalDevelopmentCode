# 📖 SECTION A — JAVA CORE & SPRING BOOT Q&A

## Java Core — Must Know Answers

### 1. What are the 4 pillars of OOP?
**Answer:** Encapsulation (hiding data via private fields + getters/setters), Inheritance (IS-A relationship, extending classes), Polymorphism (same method name, different behavior — method overloading/overriding), Abstraction (hiding implementation details via interfaces/abstract classes).

### 2. Difference between Abstract Class and Interface?
| Feature | Abstract Class | Interface |
|---------|---------------|-----------|
| Methods | Can have abstract + concrete | Java 8+: default/static methods allowed |
| Variables | Can have instance variables | Only public static final |
| Constructor | Yes | No |
| Inheritance | Single | Multiple |
| Use when | IS-A + shared state | CAN-DO contract |

### 3. What is the difference between HashMap and ConcurrentHashMap?
- **HashMap**: Not thread-safe. `null` keys allowed. Fast in single-thread.
- **ConcurrentHashMap**: Thread-safe without synchronizing entire map. No `null` keys/values.

### 4. Explain Java Memory Model — Heap vs Stack?
- **Stack**: Each thread has its own. Stores method frames, local variables, references. LIFO.
- **Heap**: Shared across threads. Stores objects and class instances. Managed by GC.
- **MetaSpace** (Java 8+): Stores class metadata. Replaced PermGen.

### 5. What is volatile keyword?
Guarantees visibility — changes to a volatile variable are immediately visible to all threads. Does NOT guarantee atomicity (use `AtomicInteger` for that).

### 6. Explain ExecutorService and thread pool?
```java
ExecutorService executor = Executors.newFixedThreadPool(5);
executor.submit(() -> System.out.println("Task running"));
executor.shutdown();
```
Thread pools avoid overhead of creating new threads. Types: Fixed, Cached, Scheduled, Single.

### 7. Checked vs unchecked exceptions?
- **Checked**: Must be handled at compile time — `IOException`, `SQLException`
- **Unchecked**: Runtime — `NullPointerException`, `ArrayIndexOutOfBoundsException`
- **Error**: JVM-level — `OutOfMemoryError` (never catch these)

### 8. Java 8 Features — must mention:
- Lambda expressions: `(a, b) -> a + b`
- Stream API: `list.stream().filter(...).map(...).collect(...)`
- Optional: prevents NullPointerException
- Default/static interface methods
- `LocalDate`, `LocalDateTime` (new Date API)

### 9. What is @FunctionalInterface?
Interface with exactly ONE abstract method. Used with lambdas. Example: `Runnable`, `Comparator`, `Predicate<T>`.

## Spring Boot — Must Know Answers

### 10. How does Spring Boot auto-configuration work?
`@SpringBootApplication` = `@Configuration` + `@EnableAutoConfiguration` + `@ComponentScan`. Auto-config reads `META-INF/spring.factories` and conditionally applies beans based on classpath.

### 11. Difference between @Component, @Service, @Repository, @Controller?
All are specializations of `@Component`. Spring detects them all via component scan.
- `@Repository`: adds exception translation (converts SQL exceptions to Spring's DataAccessException)
- `@Service`: marks business logic layer
- `@Controller`: handles HTTP requests (returns views)
- `@RestController` = `@Controller` + `@ResponseBody`

### 12. What is @Transactional and when to use it?
Ensures database operations run in a transaction. If any operation fails, everything rolls back.
```java
@Transactional
public void transferMoney(Long from, Long to, BigDecimal amount) {
    debit(from, amount);  // if this passes...
    credit(to, amount);   // ...but this fails → both rolled back
}
```
Put on **service layer methods**, not repository.

### 13. Spring Bean lifecycle:
Instantiation → Dependency Injection → `@PostConstruct` → In Use → `@PreDestroy` → Destroy

### 14. What is N+1 problem in JPA?
When fetching a list of entities with lazy-loaded collections — causes 1 query for list + N queries for each entity's collection.
**Fix:** Use `JOIN FETCH` in JPQL or `@EntityGraph`.
```java
@Query("SELECT u FROM User u JOIN FETCH u.orders WHERE u.id = :id")
```

### 15. Explain Spring Security filter chain:
Request → `DelegatingFilterProxy` → `SecurityFilterChain` → `UsernamePasswordAuthenticationFilter` → `AuthenticationManager` → `UserDetailsService` → authenticate → `SecurityContextHolder`.

### 16. How does JWT authentication work?
1. User sends username/password → server validates → creates JWT (header.payload.signature)
2. Client stores JWT (localStorage or cookie)
3. On each request, client sends `Authorization: Bearer <token>`
4. Server validates signature → extracts claims → allows/denies

```
JWT = Base64(header) + "." + Base64(payload) + "." + HMAC(secret)
```

### 17. What is @RestControllerAdvice?
Global exception handler. Catches exceptions across all controllers:
```java
@RestControllerAdvice
public class GlobalExceptionHandler {
    @ExceptionHandler(ResourceNotFoundException.class)
    public ResponseEntity<String> handleNotFound(ResourceNotFoundException ex) {
        return ResponseEntity.status(404).body(ex.getMessage());
    }
}
```

### Key Microservices Concepts:

| Concept | Your Answer |
|---------|------------|
| **Service Discovery (Eureka)** | Services register on startup. Gateway queries Eureka for instance IP instead of hardcoding. Handles multiple instances automatically. |
| **API Gateway** | Single entry point. Handles routing, auth, rate limiting, logging. Without it, client would need to know every service URL. |
| **Circuit Breaker** | Like an electrical breaker. If service fails X times → circuit opens → fallback method runs → after timeout, circuit half-opens to test recovery. |
| **Database per service** | Each microservice owns its DB. Prevents tight coupling. Order Service cannot query Product DB directly — must go through API. |
| **Eventual Consistency** | In distributed systems, data consistency happens over time. Use events (Kafka/RabbitMQ) for async communication. |
| **Saga Pattern** | Manages distributed transactions. Each service publishes event on success. If a step fails, compensating transactions run. |

# 📖 SECTION C — ANGULAR & FRONTEND

> The JD asks for HTML/CSS/JavaScript + Spring/backend. Angular is your edge.

### Angular Top 10 Q&A:

**1. What is Angular?**
TypeScript-based SPA framework by Google. Component-based architecture with two-way data binding.

**2. What is a Component?**
Building block of Angular app. Has template (HTML), styles (CSS), and logic (TypeScript class).

**3. What is a Service?**
Class marked with `@Injectable`. Handles business logic, API calls, shared state. Injected via DI.

**4. What is RxJS Observable?**
Stream of async data. Components subscribe to observables from services.
```typescript
this.userService.getUsers().subscribe(users => this.users = users);
```

**5. Promise vs Observable?**
- Promise: single value, eager, not cancellable
- Observable: stream of values, lazy, cancellable, supports operators (map, filter, switchMap)

**6. What is Angular Routing?**
`RouterModule` maps URL paths to components. `RouterLink`, `RouterOutlet` in templates.

**7. What is two-way data binding?**
`[(ngModel)]` — changes in input reflect in component and vice versa.

**8. What is Dependency Injection?**
Angular's DI system provides instances of services to components automatically. Services are singletons by default.

**9. What is a Pipe?**
Transforms displayed values. `{{ date | date:'dd/MM/yyyy' }}`. Can create custom pipes.

**10. ngOnInit vs constructor?**
Constructor: DI setup only. ngOnInit: initialization logic + API calls after component is ready.

# 🗂️ SECTION I — DATA STRUCTURES INTERNAL WORKING

> **The interview question: "What data structures did you use and how do they work internally?"**
> This is asked to test depth. Know these cold.

---

## 1. HashMap — Internal Working

**How it works internally:**
- HashMap uses an **array of buckets** (default 16). Each bucket is a linked list (Java 8+: becomes a Red-Black Tree when bucket size > 8).
- When you do `map.put(key, value)`:
  1. `hashCode()` is called on key → produces a hash
  2. Hash is mapped to a bucket index: `index = hash & (capacity - 1)`
  3. If bucket is empty → entry stored directly
  4. If bucket has entries → **collision** → entries chained as linked list
  5. To retrieve: same hash → same bucket → `.equals()` used to find exact key

**Key concepts to mention:**
- **Load factor** (default 0.75): when 75% full → HashMap **rehashes** (doubles capacity, redistributes entries)
- **hashCode() + equals()** contract: if two objects are equal, they MUST have same hashCode
- Java 8+: bucket converts to Red-Black Tree after 8 entries → O(log n) instead of O(n)

**Time complexity:**
- Average: `put/get/remove` = O(1)
- Worst case (all keys same bucket): O(n) linked list, O(log n) tree

**Interview answer script:**
> "HashMap uses an array of buckets internally. When a key-value pair is added, the key's hashCode() determines which bucket it goes into. Collisions are handled via linked lists (or Red-Black Trees in Java 8+). The load factor of 0.75 triggers rehashing when capacity is 75% full. Retrieval uses hashCode() to find the bucket, then equals() to find the exact key. Average time complexity is O(1) for get and put."

---

## 2. ArrayList — Internal Working

**How it works internally:**
- Backed by a plain **Object[] array** with a default initial capacity of 10
- When you `add()` beyond capacity → creates a **new array 1.5× larger** → copies all elements → old array garbage collected
- `get(index)` is O(1) — direct array access
- `add(index, element)` is O(n) — shifts all elements right
- `remove(index)` is O(n) — shifts elements left

**Interview answer script:**
> "ArrayList is backed by a dynamic array. It starts with capacity 10. When full, it creates a new array 1.5 times larger and copies all elements — this is called resizing. Random access by index is O(1). Insertion/deletion in the middle is O(n) due to shifting."

---

## 3. LinkedList — Internal Working

**How it works internally:**
- **Doubly linked list** — each node has `data`, `prev` pointer, `next` pointer
- No random access — to get element at index N, must traverse from head: O(n)
- `add()/remove()` at head or tail: O(1) — just update pointers
- `add()/remove()` in middle: O(n) to find position, then O(1) to relink
- More memory than ArrayList — each node stores 2 extra pointers

**When to use LinkedList vs ArrayList:**
- ArrayList: frequent reads, random access
- LinkedList: frequent insertions/deletions at head/tail

**Interview answer script:**
> "LinkedList is a doubly linked list. Each node holds data and pointers to the previous and next node. Insertion and deletion at the ends are O(1) — just pointer updates. But accessing by index is O(n) since you traverse from head. It uses more memory than ArrayList due to the extra pointers."

---

## 4. Stack — Internal Working

- Built on `Deque` (use `ArrayDeque` in Java — not legacy `Stack` class)
- LIFO — Last In First Out
- `push()` adds to top, `pop()` removes from top, `peek()` reads top without removing
- Internally: `ArrayDeque` uses a resizable circular array
- All operations O(1)

**Use cases:** function call stack, undo/redo, balanced parentheses, expression evaluation

**Interview answer:** "Stack follows LIFO. In Java, ArrayDeque is the preferred implementation. Push and pop are O(1). I used a stack in the Valid Parentheses problem — push opening brackets, pop and match on closing brackets."

---

## 5. Queue — Internal Working

- FIFO — First In First Out
- `LinkedList` or `ArrayDeque` implements `Queue` interface
- `offer()` adds to rear, `poll()` removes from front, `peek()` reads front
- `PriorityQueue`: elements ordered by natural order or Comparator — backed by a **min-heap** (binary heap array)

**Interview answer:** "Queue is FIFO. Java's ArrayDeque or LinkedList implements it. PriorityQueue uses a min-heap internally — the smallest element is always at the front. Used in BFS, task scheduling, and top-K problems."
